## 2 与大模型的对话技巧
### 2.1 表达详细
提出的要求尽可能详细。想象AI是你手底下的员工。作为员工，第一原则是尽可能花少的精力完成老板给出的任务，以争取到更多的摸鱼时间。
```txt
帮我写一个可以自动调系统PID的软件。
```

```txt
帮我写一个可以自动调系统PID的软件。要求用AI辅助调整系统参数，并且有图形化界面观察参数变化。图形化界面使用QT完成。
```

### 2.2 拆分需求
在对当前的问题有一定研究的情况下，可以用自己的知识储备去拆分这个问题，拆成几个较小的部分分别完成。
```txt
帮我完成一个电商网站，要求美观精致，主题突出，功能丰富。
```

```txt
帮我完成一个电商网站，要求美观精致，主题突出，功能丰富。你先给出前端的样式，并在我的指导下完成修改，然后再开始设计后端。
```

实际上,前两点可以总结成`Role (角色) + Context (背景) + Task (任务) + Constraint (约束)`这样一个模式.
### 2.3 思维链引导
在提示词中加入相关内容，强制让AI进行思维链推导，可以减少AI幻觉率。例如
`为我列出每一步的数学推导,并说明变换的合理性`.
PS:关于减少幻觉,也可以做交叉验证.让两个大模型互相评估对方的回答.
### 2.4 反向约束
明确告诉AI`不要做什么`，例如`不要使用类组件`、`不要引入第三方状态管理库`，减少不符合要求的输出。

### 2.5 给出示例或引导
例如，`仿照我的代码风格，给我完整代码`，`在这个例子中，答案应该输出xxx`。

### 2.6 大语言模型的原理
正如前文所言，大语言模型是一个计算概率分布的自动机。他会根据你和他对话的上下文，预测下一个字最大概率是什么，并输出这个字。而上下文在他看来是以token为单位的，一个token在大模型内部是一个高维度向量。模型会根据向量间的关系计算next token最有可能的概率。

因此我们在提示词上做的工作，本质上是在微调大模型的上下文，让他更接近我们想要的输出结果。这一切都是《Attention is all your need》中提出的理论。注意力机制的核心是计算一个词（Query）对序列中所有词（Keys）的**相似度**。

- **计算公式：** $Attention(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$
    
- **过程：** 模型把每个 token 变成向量，然后做点积（Dot Product）。点积的结果反映了两个向量在空间里的“对齐程度”。
    
- **意义：** 如果 token A 和 token B 的语义关系很紧密（比如“吃”和“苹果”），它们的点积值就大。
    

---

#### Softmax 的作用

当你看到注意力矩阵里那些 0.1、0.5 的数值时，那是经过 `Softmax` 处理后的结果：

- 它把原始的相关性得分（Logits）归一化。
    
- 这确实形成了一个概率分布，但这个“概率”的含义是：**“在处理当前这个词时，我有百分之多少的精力应该放在另一个词上。”**
    
- 这被称为**注意力权重（Attention Weights）**。
    

---

#### 真正的“概率”在哪里？

大模型中真正的“概率”通常出现在**模型的最后一步**（输出层）：

- **注意力层：** 发生在模型内部。它把所有 token 的信息进行加权融合，得到一个融合了上下文的新向量。
    
- **输出层（Linear + Softmax）：** 模型把这个融合后的向量映射到整个词表（比如 5 万个词），最后算出一个概率分布。这个分布才是在预测：**“下一个 token 是某个词的概率是多少？”**

在两三年前，gpt-3.5和4o刚刚横空出世的时候，在提示词中加入思维链引导是一个很有效的手段，可以大幅减少大模型的幻觉。如今大模型的thinking模式，其实就是把思维链隐藏在了模型内部，它可以自己推导自己要做的事情。

## 3 用AI辅助开发
结合上文所说的技巧和原理，其实我们能大致描绘出用AI辅助开发的标准流程。
- **第一步：梳理需求**：整理清晰客户需求，逐条列出功能要求，作为AI开发的输入材料。
  |> 在Readme中拆解项目需求,再列好开发步骤,期望结果,最后写一个模型开发日志记录开发进展.
    
- **第二步：搭建项目架构**：
    
    1. 将整理好的需求文档、明确的技术栈要求发给AI，添加需要用到的Skill
        
    2. AI会自动生成前端、后端目录结构和基础代码，开发者只需要review每一处修改，接受符合要求的改动，拒绝不符合要求的改动
        
    3. 手动创建空数据库，将数据库连接信息发给AI，AI会自动配置连接，生成建表语句
        
- **第三步：功能实现与调试**：
    
    - 逐个功能开发调试，遇到BUG时，🚩重点：开发者尽量先定位缩小问题范围，再将问题文件和BUG描述发给AI，可大幅减少Token消耗，降低成本，提升排错效率
        
    - 需要美化UI时，可直接调用界面设计Skill，让AI重新设计界面配色与布局
        
- **第四步：功能测试**：启动项目预览效果，验证功能是否符合要求，逐步迭代优化.或者让AI自行编写单元测试.(相比起代码,单元测试更加好理解)

在一两年前,如果想要大模型生成一些前端完善的页面,或者按需求做出改动,几乎很难做到.但现在随着模型的进步,我们今天已经可以轻而易举地用AI生成一个精美的网页,哪怕我们不懂任何前端的知识.(这也许是前端集体失业的原因)(关于AI是否会让我们失业的看法).

大模型的进化其实就是在优化他直觉上应该具备的一些能力.现在我们可以通过提示词工程来达到我们想要的目的,也许未来可以简化成一句话.
